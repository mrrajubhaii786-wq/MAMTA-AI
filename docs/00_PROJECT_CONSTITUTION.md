## 2. Core Principles

### 2.1 AI Independence
- No future development depends on one long AI conversation
- Project must continue from GitHub documentation
- AI sessions are temporary; documentation is permanent

### 2.2 Documentation First
- Every feature must be documented before coding
- Every decision must be logged
- Every sprint must be tracked

### 2.3 GitHub as Source of Truth
- GitHub repository is the only permanent record
- All code lives in GitHub
- All documentation lives in GitHub

---

## 3. Development Rules

### Rule 1: Never Ask "What Were We Doing?"
- Read documentation first
- Continue from last verified task
- Check PROGRESS_TRACKER.md

### Rule 2: Every Sprint Must Be Documented
- Sprint number, objective, tasks
- Completed, pending, blocked items
- Evidence and date

### Rule 3: Every Decision Must Be Logged
- Why was this chosen?
- What were the alternatives?
- What are the consequences?

### Rule 4: Every Feature Must Be Tested
- Local test before push
- Live test after deploy
- User verification required
